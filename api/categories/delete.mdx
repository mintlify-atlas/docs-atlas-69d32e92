<Warning>
  This endpoint is not yet implemented in the current version of the API. The documentation below represents the expected implementation.
</Warning>

Deletes a category from the system.

## Endpoint

```
DELETE /api/Categoria/Eliminar/{id}
```

## Path Parameters

<ParamField path="id" type="integer" required>
  Unique identifier of the category to delete
</ParamField>

## Response

Returns a `Response<bool>` wrapper indicating success or failure.

<ResponseField name="status" type="boolean" required>
  Indicates whether the request was successful
</ResponseField>

<ResponseField name="value" type="boolean">
  True if the category was deleted successfully, false otherwise
</ResponseField>

<ResponseField name="msg" type="string">
  Error message if status is false, null otherwise
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X DELETE "https://api.example.com/api/Categoria/Eliminar/4" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const categoryId = 4;
const response = await fetch(`https://api.example.com/api/Categoria/Eliminar/${categoryId}`, {
  method: 'DELETE',
  headers: {
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data);
```

```csharp C#
using var client = new HttpClient();
var categoryId = 4;
var response = await client.DeleteAsync(
  $"https://api.example.com/api/Categoria/Eliminar/{categoryId}"
);
var content = await response.Content.ReadAsStringAsync();
```

</CodeGroup>

## Example Response

```json Success Response
{
  "status": true,
  "value": true,
  "msg": null
}
```

```json Error Response
{
  "status": false,
  "value": false,
  "msg": "Cannot delete category: products are associated with this category"
}
```

## Response Codes

| Code | Description |
|------|-------------|
| 200  | Category deleted successfully |
| 400  | Cannot delete category (may have dependencies) |
| 404  | Category not found |
| 500  | Internal server error |

## Notes

- Deleting a category that has associated products may be prevented to maintain referential integrity
- Consider implementing soft deletes to preserve historical data
