## Endpoint

```
GET /api/Producto/Lista
```

Retrieves all products with their associated category information, stock levels, and pricing.

## Response

The response is wrapped in a `Response<T>` object that includes status information.

<ResponseField name="status" type="boolean" required>
  Indicates whether the request was successful
</ResponseField>

<ResponseField name="value" type="array" required>
  Array of product objects
  
  <Expandable title="ProductoDTO properties">
    <ResponseField name="idProducto" type="integer">
      Unique identifier for the product
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Product name
    </ResponseField>
    
    <ResponseField name="idCategoria" type="integer">
      ID of the category this product belongs to
    </ResponseField>
    
    <ResponseField name="descripcionCategoria" type="string">
      Category name/description
    </ResponseField>
    
    <ResponseField name="stock" type="integer">
      Current stock quantity
    </ResponseField>
    
    <ResponseField name="precio" type="string">
      Product price (stored as string)
    </ResponseField>
    
    <ResponseField name="esActivo" type="integer">
      Active status (1 = active, 0 = inactive)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="msg" type="string">
  Error message (only populated when status is false)
</ResponseField>

## Example Request

<CodeGroup>

## Response Status

This endpoint always returns HTTP 200. Check the `status` field in the response body to determine success or failure.

- `status: true` - Products retrieved successfully
- `status: false` - An error occurred (check `msg` field for details)
